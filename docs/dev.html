Key Rules (Microsoft Project / VBA)

These rules avoid common runtime issues when working inside an .mpt environment.

Project Application Object

Never use (inside .mpt):

Dim pjApp
Set pjApp = CreateObject("MSProject.Application")  ' ❌ Avoid

Always prefer:

Dim pjApp As Application
Set pjApp = Application  ' ✅ Use host application

General Conventions

  • Scope macros to modules in /macros/production/ only for shipped features.
  • Keep UI-related customizations (e.g., Ribbon XML) versioned; apply via scripts (e.g., add_ribbon_to_mpt.ps1).
  • Use Option Explicit and explicit types in all modules.
  • Document any Project options or trust settings required to run macros.
  • Follow Conventional Commits for changes: feat:, fix:, docs:, chore:, etc.
