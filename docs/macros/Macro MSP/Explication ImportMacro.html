MISSION : Tu as reçu un fichier qui se termine par .bas. Tu veux l'utiliser dans MS Project. Tu vas apprendre à l'importer, à vérifier qu'il fonctionne, et à créer un bouton pour t'en servir quand tu veux.

ÉTAPE 1 : Prépare ton fichier .bas

D'abord, assure-toi que tu as bien téléchargé le fichier. Il doit se terminer par .bas et porter un nom simple comme calculateur.bas ou macro_planning.bas. Place-le dans un endroit que tu retrouves facilement, par exemple ton Bureau ou tes Téléchargements.

ATTENTION : Si Windows a ajouté .txt à la fin, renomme le fichier pour enlever cette extension. Il doit vraiment finir par .bas uniquement.

ÉTAPE 2 : Ouvre MS Project

Lance Microsoft Project. Ouvre un projet existant ou crée-en un nouveau, peu importe. L'important c'est d'avoir MS Project ouvert et fonctionnel devant toi.

ÉTAPE 3 : Accède à l'éditeur VBA

Maintenant, tu vas accéder à la partie cachée de MS Project où vivent les macros. Appuie simultanément sur les touches ALT et F11. Une nouvelle fenêtre va s'ouvrir. Cette fenêtre s'appelle l'éditeur VBA. Elle peut paraître intimidante mais c'est normal.

Si rien ne se passe quand tu appuies sur ALT+F11, essaie de passer par le menu : Fichier, puis Options, puis Centre de gestion de la confidentialité, puis Paramètres du Centre de gestion, puis Paramètres des macros, et assure-toi que les macros ne sont pas complètement désactivées.

ÉTAPE 4 : Importe ton fichier .bas

Dans l'éditeur VBA qui vient de s'ouvrir, va dans le menu Fichier en haut à gauche. Clique dessus, puis cherche et clique sur Importer un fichier. Une fenêtre de navigation va s'ouvrir.

Navigue jusqu'à l'endroit où tu as mis ton fichier .bas. Sélectionne-le en cliquant dessus une fois, puis clique sur le bouton Ouvrir.

Si tu ne vois pas ton fichier .bas dans la liste, vérifie que le filtre en bas de la fenêtre est réglé sur "Tous les fichiers" ou "Fichiers BASIC (*.bas)".

ÉTAPE 5 : Vérifie que l'import a fonctionné

Après l'import, regarde dans la partie gauche de l'éditeur VBA. Tu dois voir apparaître le nom de ton fichier dans une liste qui s'appelle "Explorateur de projets". Si tu vois ton fichier listé là, double-clique dessus. Du code doit apparaître dans la partie droite de l'écran.

PROBLÈME FRÉQUENT :

Si tu ne vois rien apparaître dans l'explorateur de projets, l'import a échoué. Recommence l'étape 4 en vérifiant bien le nom et l'emplacement de ton fichier .bas.

ÉTAPE 6 : Ferme l'éditeur VBA

Une fois que tu as confirmé que ton code est bien importé, ferme l'éditeur VBA en cliquant sur la croix rouge en haut à droite de sa fenêtre. Tu retournes ainsi dans MS Project normal.

ÉTAPE 7 : Crée un bouton pour exécuter ta macro

Maintenant tu vas créer un bouton facile d'accès pour utiliser ta macro. Dans MS Project, fais un clic droit n'importe où sur le ruban (la barre d'outils en haut). Dans le menu qui apparaît, clique sur Personnaliser le ruban.

Une fenêtre de paramètres va s'ouvrir. Dans la partie droite, tu vois une liste d'onglets. Sélectionne un onglet existant ou crée-en un nouveau en cliquant sur Nouvel onglet.

Ensuite, dans cet onglet, clique sur Nouveau groupe pour créer un groupe de boutons.

Maintenant, dans la partie gauche de la fenêtre, change le menu déroulant pour sélectionner Macros. Tu dois voir apparaître le nom de ta macro dans la liste. Sélectionne-la, puis clique sur Ajouter pour l'ajouter à ton groupe.

Clique sur OK pour valider ces changements.

ÉTAPE 8 : Test final

Tu dois maintenant voir un nouveau bouton dans le ruban de MS Project. Clique dessus pour tester ta macro. Si tout fonctionne, félicitations ! Si ça ne marche pas, lis la section erreurs ci-dessous.

ERREURS COURANTES ET SOLUTIONS :

Le bouton n'apparaît pas : Tu n'as pas correctement ajouté la macro au ruban. Recommence l'étape 7 en vérifiant chaque action.

Le bouton ne fait rien quand je clique : Soit la macro n'est pas correctement importée (revérifie l'étape 5), soit MS Project bloque l'exécution des macros. Va dans Fichier, Options, Centre de gestion de la confidentialité, et autorise les macros.

Message "Macro introuvable" : Le nom que tu as donné au bouton ne correspond pas exactement au nom de la macro dans le code. Vérifie les noms et recommence l'ajout du bouton.

L'éditeur VBA ne s'ouvre pas : Les macros sont peut-être complètement désactivées sur ton système. Demande à ton administrateur informatique ou consulte les paramètres de sécurité de MS Project.

CONSEIL FINAL :

Sauvegarde ton projet MS Project après avoir importé la macro. Comme ça, la prochaine fois que tu ouvriras ce projet, ta macro sera toujours disponible. Si tu veux que la macro soit disponible dans tous tes projets, il faudra l'ajouter au modèle global de MS Project, mais c'est une procédure plus avancée.
