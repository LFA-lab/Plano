Version 4.0 - Analyse Hybride pour Pilotage Temps Réel Optimal

Date : 18/08/2025 | Fichier : ExportHeuresProjet.bas

Objectif

Cette macro génère un export Excel spécialement conçu pour les dirigeants de Vinci Energies & Construction, permettant une lecture en 1 minute maximum de la santé globale d'un projet.

Innovation Clé : Distinction Confirmé vs Prévisionnel

La macro différencie les gains/pertes confirmés (tâches 100% terminées) des gains/pertes prévisionnels (tâches en cours), offrant une vision hybride pour optimiser le pilotage temps réel.

Structure de l'Export

L'export génère 3 feuilles Excel :

RESUME_DIRIGEANT

Vue synthétique en 1 minute avec KPI globaux, Top 3 lots critiques, gains/pertes confirmés et prévisionnels

PROJET

Analyse détaillée par lot avec structure arborescente et indicateurs de performance avancés

RECAP_LOTS

Tableau de bord complet avec tous les lots, filtres et métriques de pilotage

Prérequis Techniques

  1. MS Project ouvert avec un projet contenant des tâches avec heures planifiées et réalisées
  2. Date d'état définie dans MS Project (StatusDate) - sinon la date du jour sera utilisée
  3. Excel installé sur le poste de travail
  4. Données cohérentes : Tâches avec % d'avancement, heures budgétées (Work) et réalisées (ActualWork)

Utilisation

  1. Ouvrir votre projet MS Project
  2. Vérifier que la date d'état est correcte (Projet > Informations sur le projet > Date d'état)
  3. Exécuter la macro ExportHeuresProjet()
  4. Choisir le répertoire de sauvegarde dans la boîte de dialogue
  5. L'export Excel s'ouvre automatiquement avec les 3 feuilles

KPI et Métriques Calculés

KPI Globaux du Projet

Indicateur Formule Objectif Interprétation
SPI Global ΣEV / ΣPV ≥ 0.90 Dans les temps / Retard modéré / Retard critique
CPI Global ΣEV / ΣAC ≥ 0.90 Budget OK / Surcoût modéré / Surcoût critique
Gains Confirmés Σ(Gains tâches 100%) > 0 Heures économisées validées
Pertes Confirmées Σ(Pertes tâches 100%) = 0 Heures perdues validées

Métriques par Lot

  • Gains/Pertes Prévisionnels : Estimations basées sur l'avancement des tâches en cours
  • Fiabilité : % de tâches avec données cohérentes (AC > 0 ET %C > 0)
  • Écart Heures : Différence entre heures planifiées et réalisées
  • SPI/CPI par Lot : Indicateurs de performance individuels

Codes Couleur Automatiques

  • Vert : Performance satisfaisante (SPI/CPI ≥ 0.90)
  • Orange : Vigilance requise (0.80 ≤ SPI/CPI < 0.90)
  • Rouge : Action immédiate (SPI/CPI < 0.80)
  • Gris : Données non fiables (fiabilité < 50%)

Limitations et Bonnes Pratiques

Limitations Connues

  • Les tâches sans heures budgétées (Work = 0) sont exclues de l'analyse
  • Les lots avec fiabilité < 50% sont marqués comme "données non fiables"
  • L'analyse prévisionnelle nécessite un avancement minimum de 20%

Bonnes Pratiques

  • Mise à jour régulière : Exécuter l'export chaque semaine après mise à jour des % d'avancement
  • Cohérence des données : Vérifier que les heures réalisées sont bien saisies
  • Focus TOP 3 : Concentrer les actions correctives sur les 3 lots les plus critiques
  • Validation des gains : Ne considérer comme acquis que les gains confirmés (tâches 100%)
